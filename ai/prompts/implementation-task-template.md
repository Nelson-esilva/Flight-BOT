# Template: tarefa de implementacao

## Objetivo

Descreva uma funcionalidade pequena e verificavel.

## Arquivos permitidos

- Liste caminhos especificos que podem mudar.

## Arquivos proibidos

- Liste caminhos que nao devem mudar.

## Criterios de aceite

- Comportamento esperado.
- Entrada e saida esperadas.
- Casos de erro relevantes.

## Testes esperados

- Testes unitarios.
- Testes de integracao, se aplicavel.
- Comandos de validacao.

## Restricoes

- Nao criar arquivos fora do escopo.
- Nao duplicar responsabilidade existente.
- Nao implementar funcionalidade futura.
- Nao inventar comportamento de API externa.
